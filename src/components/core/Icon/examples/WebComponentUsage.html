Icon Web Component Usage Examples

This page demonstrates various ways to use the <dp-icon> web component. The Icon component provides a flexible, accessible way to display SVG icons with support for multiple sizes, color inheritance, and programmatic manipulation.

1. Basic Icon Usage

Simple icon with default settings (24px, inherits color)

<!-- Default icon (24px, inherits parent color) --> <dp-icon name="circle"></dp-icon> <dp-icon name="check"></dp-icon> <dp-icon name="x"></dp-icon> <dp-icon name="heart"></dp-icon> <dp-icon name="settings"></dp-icon>

2. All Available Icons

3. Size Variants

Icons at different sizes (16px, 24px, 32px, 40px)

<!-- Size variants --> <dp-icon name="heart" size="16"></dp-icon> <!-- 16px --> <dp-icon name="heart" size="24"></dp-icon> <!-- 24px (default) --> <dp-icon name="heart" size="32"></dp-icon> <!-- 32px --> <dp-icon name="heart" size="40"></dp-icon> <!-- 40px -->

4. Color Inheritance

Icons automatically inherit color from parent element

Blue text with blue icon (color inherited from parent)
Green text with green icon (color inherited from parent)
Red text with red icon (color inherited from parent)
Purple text with purple icon (color inherited from parent)
<!-- Icons inherit color from parent element --> <div style="color: #3b82f6;"> <dp-icon name="check"></dp-icon> <span>Blue text with blue icon</span> </div> <div style="color: #10b981;"> <dp-icon name="check"></dp-icon> <span>Green text with green icon</span> </div>

5. Explicit Color Override

Override inherited color with explicit color value

<!-- Override color with inline style --> <dp-icon name="heart" style="color: #ef4444;"></dp-icon> <dp-icon name="heart" style="color: #f59e0b;"></dp-icon> <dp-icon name="heart" style="color: #10b981;"></dp-icon> <dp-icon name="heart" style="color: #3b82f6;"></dp-icon> <dp-icon name="heart" style="color: #8b5cf6;"></dp-icon>
Note: While you can override color with inline styles, it's recommended to use color inheritance from parent elements for better maintainability and consistency with your design system.

6. Programmatic Manipulation

Create and modify icons using JavaScript

<script> // Create icon programmatically const icon = document.createElement('dp-icon'); icon.name = 'heart'; icon.size = 32; icon.style.color = '#ef4444'; document.body.appendChild(icon); // Modify existing icon icon.name = 'star'; icon.size = 40; icon.setAttribute('test-id', 'my-icon'); // Access properties console.log(icon.name); // 'star' console.log(icon.size); // 40 console.log(icon.testID); // 'my-icon' </script>

7. Icons in Buttons and UI Elements

Common use cases with buttons and interactive elements

<!-- Icons in buttons --> <button> <dp-icon name="plus" size="16"></dp-icon> Add Item </button> <button> <dp-icon name="check" size="16"></dp-icon> Save </button> <button> Settings <dp-icon name="settings" size="16"></dp-icon> </button>

8. Accessibility

Icons are decorative and hidden from screen readers

<!-- Icons are automatically aria-hidden="true" --> <!-- Button text provides accessible label --> <button> <dp-icon name="check" size="16"></dp-icon> <span>Confirm</span> </button>
Accessibility Note: Icons are automatically marked with aria-hidden="true" because they are decorative. Always provide text labels for interactive elements that use icons.
