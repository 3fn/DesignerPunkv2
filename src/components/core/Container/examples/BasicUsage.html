1. Container with Minimal Props

Minimal Container: Container with no styling props renders as an unstyled wrapper. This is useful when you need a semantic container without visual styling.

This is a Container with no styling props. It renders as a simple wrapper with no visual styling applied.
<dp-container> <p>Content goes here</p> </dp-container>

2. Container with Children

Container with Padding: Adding padding creates internal spacing around the content. This example uses padding="200" which maps to 16px (space.inset.200 token).

Container with Children

This Container has padding="200" and background="color.surface". The padding creates comfortable internal spacing, and the background provides visual separation from the page.

Multiple child elements can be nested inside the Container, and they will all be wrapped with the same styling.
<dp-container padding="200" background="color.surface"> <h3>Container with Children</h3> <p>First paragraph</p> <p>Second paragraph</p> </dp-container>

3. Common Prop Combinations

Card-like Container: Combining padding, background, shadow, and borderRadius creates a card-like appearance. This is a common pattern for content containers.

Card-like Container

This Container combines multiple styling props:

  • padding="300" (24px internal spacing)
  • background="color.surface" (surface color)
  • shadow="shadow.container" (subtle elevation shadow)
  • border-radius="normal" (8px rounded corners)

This combination creates a visually distinct container that appears elevated from the page.
<dp-container padding="300" background="color.surface" shadow="shadow.container" border-radius="normal" > <h3>Card Title</h3> <p>Card content</p> </dp-container>
Panel-like Container: Using border instead of shadow creates a different visual style. This pattern is useful for sections that need clear boundaries without elevation.

Panel-like Container

This Container uses a border instead of a shadow:

  • padding="200" (16px internal spacing)
  • background="color.surface" (surface color)
  • border="default" (1px border)
  • border-radius="tight" (4px rounded corners)

This creates a flat, bordered container without elevation shadow.
<dp-container padding="200" background="color.surface" border="default" border-radius="tight" > <h3>Panel Title</h3> <p>Panel content</p> </dp-container>
Compact Container: Using smaller padding values creates a more compact container. This is useful for UI elements that need minimal spacing.

Compact container with padding="100" (8px). Useful for tight UI elements like tags, badges, or compact cards.
<dp-container padding="100" background="color.surface" border-radius="tight" > <p>Compact content</p> </dp-container>
Spacious Container: Using larger padding values creates a more spacious container. This is useful for hero sections or prominent content areas.

Spacious Container

This Container uses padding="400" (32px) for generous internal spacing. This creates a more prominent, spacious feel suitable for hero sections or featured content.
<dp-container padding="400" background="color.surface" shadow="shadow.container" border-radius="normal" > <h3>Hero Title</h3> <p>Hero content</p> </dp-container>

4. Semantic HTML Elements

Semantic Article Container: Using semantic="article" renders the Container as an HTML article element. This improves accessibility and SEO by providing semantic meaning to the content structure.

Article Container

This Container uses semantic="article" to render as an HTML <article> element. This provides semantic meaning for screen readers and search engines.

The accessibility-label prop adds an aria-label attribute for additional context.
<dp-container semantic="article" padding="300" background="color.surface" border-radius="normal" accessibility-label="Example article" > <h3>Article Title</h3> <p>Article content</p> </dp-container>
