## Overview

Conducted 4 design exercises with Leonardo and Peter against the Working Class civic engagement app. Used Application MCP tools (find_components, get_experience_pattern, validate_assembly) to select components, validate assemblies, and identify gaps across real product screens. All 7 production families exercised across 4 screen types: data capture (profile), content consumption (legislation feed), content list (notifications), and overview (dashboard).

## Key Findings

- **12 missing experience patterns** identified, all tagged `general` — most impactful: dashboard layout, notification list, multi-section form
- **3 component gaps**: content list item (highest priority), linear progress bar, Container-Card-Base readiness (needed in 3/4 exercises)
- **4 MCP tool gaps**: `purpose` search blind spots, `context`/`purpose` don't cross-reference, `dashboards` context underserved, pattern naming inconsistency
- **6 design principles** surfaced, including "What is → What will be → What was" dashboard hierarchy
