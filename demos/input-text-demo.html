9. Accessibility Features

All Input-Text variants are built with WCAG 2.1 AA compliance as a first-class concern.

Keyboard Navigation

Use Tab to move between inputs. Disabled inputs are skipped. Focus ring is visible on keyboard navigation:

ARIA Attributes

  • aria-describedby ‚Äî links input to helper text and error message elements
  • aria-invalid="true" ‚Äî set when error-message is present
  • role="alert" ‚Äî on error message element for screen reader announcement
  • required ‚Äî native HTML required attribute for form validation
  • disabled ‚Äî native HTML disabled attribute, removes from tab order

Reduced Motion

prefers-reduced-motion: When the user has reduced motion enabled, the float label animation is replaced with an instant transition (no animation). All focus transitions also respect this preference.

13. Usage Examples

HTML

<!-- Basic text input -->
<input-text-base id="name" label="Full name" value=""></input-text-base>

<!-- With helper text and placeholder -->
<input-text-base
  id="bio"
  label="Bio"
  placeholder="Tell us about yourself"
  helper-text="Max 200 characters"
  max-length="200"
></input-text-base>

<!-- Error state -->
<input-text-base
  id="email"
  label="Email"
  value="bad-email"
  error-message="Please enter a valid email"
></input-text-base>

<!-- Email variant (validates on blur) -->
<input-text-email
  id="email"
  label="Email address"
  placeholder="you@example.com"
  required
></input-text-email>

<!-- Password variant (with requirements) -->
<input-text-password
  id="password"
  label="New password"
  is-new-password
  min-length="8"
  require-uppercase
  require-number
  require-special-char
></input-text-password>

<!-- Phone number variant -->
<input-text-phone-number
  id="phone"
  label="Phone number"
  placeholder="(555) 123-4567"
  required
></input-text-phone-number>

JavaScript

// Wait for components to be defined
await customElements.whenDefined('input-text-base');
await customElements.whenDefined('input-text-email');

// Listen for change events
const input = document.querySelector('input-text-base');
input.addEventListener('change', (event) => {
  console.log('Value changed:', event.detail.value);
});

// Listen for focus/blur
input.addEventListener('focus', () => console.log('Focused'));
input.addEventListener('blur', () => console.log('Blurred'));

// Email validation events
const email = document.querySelector('input-text-email');
email.addEventListener('validate', (event) => {
  const { isValid, errorMessage } = event.detail;
  console.log('Email valid:', isValid, errorMessage);
});

// Programmatic validation
const result = email.validate();
console.log(result.isValid, result.errorMessage);

// Password toggle events
const password = document.querySelector('input-text-password');
password.addEventListener('toggle-visibility', (event) => {
  console.log('Password visible:', event.detail.visible);
});

// Phone number change (includes formatted + raw)
const phone = document.querySelector('input-text-phone-number');
phone.addEventListener('change', (event) => {
  console.log('Formatted:', event.detail.formattedValue);
  console.log('Raw digits:', event.detail.rawDigits);
});
