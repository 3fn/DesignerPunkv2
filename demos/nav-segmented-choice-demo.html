🧭 Nav-SegmentedChoice-Base Demo

Navigation control for switching between mutually exclusive content views — sliding indicator, keyboard navigation, and animated transitions

1. Basic Usage

Segmented controls with 2 and 3 segments. Click a segment to select it. The indicator slides to the active segment.

2 segments
3 segments
4 segments — pre-selected "Month"
Noop on active: Clicking the already-selected segment does nothing — no event, no animation. This is the interaction_noop_active contract.

2. Size Variants

Standard (default) and condensed sizes. Condensed uses smaller padding and font size.

Standard (default)
Condensed

3. Keyboard Navigation

Click a segment to focus the control, then use keyboard:

  • ← / → — Move between segments (wraps around)
  • Home — Jump to first segment
  • End — Jump to last segment
  • Enter / Space — Activate focused segment

Use arrow keys after focusing a segment

Roving tabindex: Only the active segment is in the tab order (tabindex="0"). Arrow keys move focus between segments. This follows the WAI-ARIA Tabs pattern.

5. Accessibility Features

ARIA Attributes

  • role="tablist" on the container
  • role="tab" on each segment
  • aria-selected="true|false" on each segment
  • aria-controls — optional panel association via id prop

Focus Indicators

Focus ring: Uses :focus-visible for keyboard-only focus indicators. The outline uses accessibility.focus.width (2px) and accessibility.focus.color tokens with a 2px offset.

Reduced Motion

Animation suppression: When prefers-reduced-motion: reduce is active, the sliding indicator snaps instantly instead of animating. First render always snaps (no animation on initial load).

6. Token Verification

Check browser DevTools to verify these CSS custom properties are loaded:

Color Tokens

  • --color-structure-surface — container background
  • --color-structure-canvas — active indicator background
  • --color-structure-border — container border
  • --color-action-navigation — active segment text
  • --color-text-default — inactive segment text

Spacing Tokens

  • --space-inset-050 — container padding
  • --space-inset-100 — segment block padding (standard)
  • --space-inset-075 — segment block padding (condensed)

Shape & Border Tokens

  • --radius-normal — container and indicator border radius
  • --border-default — container border width

Motion Tokens

  • --motion-duration-150 — indicator slide duration
  • --motion-easing-standard — indicator slide easing

Shadow Tokens

  • --shadow-navigation-indicator — active indicator shadow

7. Usage Examples

HTML

<nav-segmented-choice
  segments='[{"value":"daily","label":"Daily"},{"value":"weekly","label":"Weekly"}]'
  selected-value="daily">
</nav-segmented-choice>

<!-- Condensed size -->
<nav-segmented-choice
  segments='[{"value":"a","label":"A"},{"value":"b","label":"B"},{"value":"c","label":"C"}]'
  selected-value="a"
  size="condensed">
</nav-segmented-choice>

JavaScript

const nav = document.querySelector('nav-segmented-choice');

// Listen for selection changes
nav.addEventListener('selection-change', (event) => {
  console.log('Selected:', event.detail.value);
});

// Programmatic selection
nav.selectedValue = 'weekly';

// Update segments dynamically
nav.segments = [
  { value: 'a', label: 'Option A' },
  { value: 'b', label: 'Option B' },
];
