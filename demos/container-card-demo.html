Specialized card container with opinionated defaults, curated props, and built-in interactive behavior

1. Overview

Container-Card-Base is a type primitive that composes Container-Base internally, exposing a curated subset of props appropriate for card use cases. It provides zero-config rendering with sensible defaults ‚Äî a good-looking card without any configuration.

Zero-Config Card

Zero-Config Card

No props needed. Uses default padding (150), surface.primary background, container shadow, and normal radius.

2. Background Variants

6. Interactive Cards

When interactive="true", cards gain hover feedback (8% darker), press feedback (12% darker), focus ring, and keyboard activation. The role prop controls ARIA semantics and keyboard behavior.

9. Container-Base vs Card-Base

Card-Base provides opinionated defaults and a curated API. Container-Base is the unconstrained primitive.

Container-Base

Requires explicit props for every visual property. No shadow by default.

 Container-Base (manual config)

Card-Base

Zero-config: shadow, background, padding, and radius all default to card-appropriate values.

 Card-Base (zero-config)
When to use which: Use Container-Base for generic layout wrapping. Use Container-Card-Base when you want a visually distinct, elevated content section with card semantics.

10. Token Verification

Check browser DevTools to verify these CSS custom properties are loaded:

Spacing Tokens (Padding)

  • --space-inset-050 ‚Äî padding 050 (4px, vertical only)
  • --space-inset-100 ‚Äî padding 100 (8px)
  • --space-inset-150 ‚Äî padding 150 (12px, default)
  • --space-inset-200 ‚Äî padding 200 (16px)

Color Tokens

  • --color-surface-primary ‚Äî primary surface (default background)
  • --color-surface-secondary ‚Äî secondary surface
  • --color-surface-tertiary ‚Äî tertiary surface
  • --color-border-default ‚Äî default border color
  • --color-structure-border-subtle ‚Äî subtle border color

Shadow Tokens

  • --shadow-container ‚Äî container elevation shadow (default)

Radius Tokens

  • --radius-100 ‚Äî normal radius (8px, default)
  • --radius-200 ‚Äî loose radius (16px)

Interaction Tokens

  • --blend-hoverDarker ‚Äî hover feedback (8% darker)
  • --blend-pressedDarker ‚Äî press feedback (12% darker)
  • --motion-focus-transition-duration ‚Äî focus transition duration
  • --motion-focus-transition-easing ‚Äî focus transition easing

11. Usage Examples

HTML

<!-- Zero-config card -->
<container-card-base>
  <p>Card content</p>
</container-card-base>

<!-- Interactive card -->
<container-card-base
  interactive="true"
  accessibility-label="View details"
>
  <h3>Card Title</h3>
  <p>Card description</p>
</container-card-base>

<!-- Interactive link card -->
<container-card-base
  interactive="true"
  role="link"
  accessibility-label="Read article"
>
  <h3>Article Title</h3>
  <p>Article excerpt...</p>
</container-card-base>

<!-- Custom styling -->
<container-card-base
  padding="200"
  background="surface.secondary"
  border="default"
  border-color="border.subtle"
  border-radius="loose"
>
  <p>Styled card content</p>
</container-card-base>

<!-- Asymmetric padding (image-bleeding card) -->
<container-card-base
  padding-block-start="none"
  padding-horizontal="150"
  padding-block-end="150"
>
  <img src="hero.jpg" alt="Hero image" />
  <p>Card content below image</p>
</container-card-base>

<!-- Semantic HTML -->
<container-card-base semantic="article">
  <h2>Blog Post Title</h2>
  <p>Blog post content...</p>
</container-card-base>

JavaScript

// Programmatic creation
const card = document.createElement('container-card-base');
card.setAttribute('padding', '200');
card.setAttribute('background', 'surface.secondary');
card.innerHTML = '<p>Dynamic card content</p>';
document.body.appendChild(card);

// Interactive card with event handling
const interactiveCard = document.createElement('container-card-base');
interactiveCard.setAttribute('interactive', 'true');
interactiveCard.setAttribute('accessibility-label', 'View details');
interactiveCard.innerHTML = '<h3>Clickable Card</h3>';

// Set onPress callback
interactiveCard.onPress = () => {
  console.log('Card pressed!');
};

document.body.appendChild(interactiveCard);
